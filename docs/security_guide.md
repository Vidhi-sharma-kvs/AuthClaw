# AuthClaw Security Guide

## RBAC

The backend permission matrix is generated by `services/rbac_matrix.py` and exposed to tenant administrators at `/security/rbac/matrix`.

Production enforcement is active when either:

- `AUTHCLAW_ENV=production`
- `AUTHCLAW_ENABLE_RBAC_ENFORCEMENT=true`

## Tenant Isolation

Run:

```bash
python scripts/tenant_isolation_report.py --fail-on-gaps
```

Every tenant table must have `tenant_id`, RLS enabled, RLS forced, and at least one tenant isolation policy. Global/system tables require explicit exemption reasons.

## Secrets

AuthClaw supports local AES-GCM, AWS Secrets Manager/KMS, and HashiCorp Vault. Production must use a managed backend.

Run:

```bash
python scripts/production_readiness_report.py --fail-on-code-gaps
```

Live KMS/Vault connectivity and rotation drills remain environment tasks.
